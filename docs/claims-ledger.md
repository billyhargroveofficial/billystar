# Shadowpipe claims ledger

Дата среза: 2026-07-16. Этот файл — реестр проверяемых утверждений, а не список желаемых свойств. Слово «verified» всегда относится только к явно указанной области наблюдения.

## Шкала доказательности

| Tier | Что означает | Чего не доказывает |
|---|---|---|
| **E0** | Идея, вывод из аналогии или непререгистрированная гипотеза | Реализуемость и эффект на реальном censor path |
| **E1** | Unit/integration test, VM/netem, эмулятор цензора | Поведение TSPU, конкретного оператора или публичного облака |
| **E1F** | Одно авторизованное bounded field observation на одном пути/операторе/окне без репликации | Repeatability даже в том же stratum, причинность и любую экстраполяцию |
| **E2** | Повторяемое наблюдение на одном реальном пути, операторе и временном окне | Универсальность по операторам, регионам, ASN и времени |
| **E3** | Пререгистрированный paired crossover на нескольких путях/операторах/датах | Долговечность против адаптирующегося цензора |
| **E4** | Независимая репликация и продольное наблюдение | Информационно-теоретическую неотличимость или вечную устойчивость |

Статусы: `V` — подтверждено в своей области; `H` — гипотеза; `C` — противоречит данным в указанной области; `S` — scope-limited; `X` — superseded. Сочетание вроде `V/S` нормально. Исторические строки не удаляются: изменение знания оформляется новой строкой и ссылкой `supersedes`.

## 16 KiB / trigger-unit ledger

Текущая рабочая модель называется **`T0 / UNKNOWN_CONDITIONAL`**: единица и предикат срабатывания не установлены. Это не синоним «per-5-tuple». Возможные конкурирующие модели — 5-tuple, destination IP/port, prefix/ASN, классифицированный flow, операторская политика, временное правило или их конъюнкция.

| ID | Утверждение | Статус / tier | Основание и область | Фальсификатор / следующий гейт |
|---|---|---|---|---|
| `TRG-001` | Любой raw TCP flow RU→foreign-DC замерзает примерно после 15–20 KiB. | `C/S`, E1F with within-run replication | `planeb-01`: 155/155 flows завершили nominal 2 MiB targets; observed counters summed to 328.409 MB (about 313.2 MiB) versus 310 MiB nominal payload, K=1…16, один RU→NL путь и короткое окно. Это один run window, не независимая temporal replication. | Уже фальсифицировано для этого IP/порта/пути/окна. Не переносить вывод на TLS/443 или другого оператора. |
| `TRG-002` | Shadowpipe carrier на том же пути замерзает около 16 KiB. | `C/S`, E1F-mixed | `planeb-02`: до отказа прошло примерно 0.8–2.5 MB (decimal). Отказы совпали с correctness faults (`AEAD`, EOF, stall), а не с единым порогом; run не идентифицирует censor cause. | Повторить после устранения correctness faults; отделить transport error от silent degradation по pcap/TCP_INFO/тайм-аутам. |
| `TRG-003` | Chrome-TLS на проверенном RU→NL пути имеет per-flow 16 KiB ceiling. | `C/S`, E1F unsealed | Две manual Chrome-TLS сессии по 20 MiB без stall, примерно 3–11 MiB/s; raw sealed bundle не сохранён. | Повторить по датам, операторам, портам и ASN с committed raw aggregates; положительный контроль должен уметь воспроизвести stall. |
| `TRG-004a` | Repo evidence доказывает trigger unit как per-5-tuple. | `C`, E1F + logic | На проверенном scope сам trigger не возник; reset unit там не идентифицируется. Старые заметки, называющие per-5-tuple «доказанным фактом», переутверждают данные. | Любой unit claim требует сначала положительно воспроизвести trigger. |
| `TRG-004b` | В trigger-active stratum новый 5-tuple получает независимый budget. | `H`, E0 / untested | Текущие отрицательные controls эту условную модель ни подтверждают, ни фальсифицируют. | Эксперимент `001-trigger-unit`: после reproduction различить fresh tuple, destination, prefix/ASN, classifier и time models. |
| `TRG-005` | Текущая лучшая модель — `T0 / UNKNOWN_CONDITIONAL`; blanket volume rule не поддержан repo evidence. | `H`, E1F-mixed | Консервативная модель совместима с тремя scoped отрицательными наблюдениями разного качества и внешними reports; она не является E2 identification result. | Положительный пререгистрированный crossover, который устойчиво выбирает одну trigger model; независимая репликация. |
| `TRG-006` | Логический mux/H2 streams внутри одного TCP создают новые 5-tuples. | `C`, E1 + network semantics | Все logical streams делят один transport connection; fragmentation/mux не меняют TCP 5-tuple. | Только отдельные transport connections могут проверить reset-by-connection; это всё равно не доказывает reset unit. |
| `TRG-007` | TCP rotation volume guard сейчас является полезной защитой по умолчанию. | `C/S`, E1F unsealed | Guard выключен по умолчанию; target trigger не воспроизвёлся, а один manual run показал примерно три порядка throughput loss при rebuild TLS+PQ. | Положительно воспроизвести target stall, затем показать preregistered benefit с ограниченным overhead и без handshake-rate tell в sealed runs. |

## Правило обновления

Новый результат добавляет: experiment ID, git SHA, manifest hash, UTC window, topology class, operator/path stratum, payload floor, fixed observation horizon, complete/stall/error taxonomy и ссылку на raw aggregate. No-target-event на fixed horizon отмечается как right-censored; payload completion и implementation failures учитываются как отдельные terminal/competing causes, а не автоматическое non-informative censoring. Если эксперимент не может отличить censor action от implementation fault, его verdict — `INCONCLUSIVE`, а не `BLOCKED` и не `PASSED`.
